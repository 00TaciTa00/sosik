# 업데이트 노트 자동 생성 — 데스크톱 앱 기획서 v4

> 작성일: 2026-04-06

---

## 전체 플로우

```bash
앱 실행
→ 등록된 레포지토리 목록 로드
→ 각 레포별 새 커밋 동시 체크
→ 새 커밋 감지된 레포에 토스트 알림
→ 레포 선택
→ git diff 추출 (레포별 보안 파일 제외 규칙 적용)
→ AI 요약 생성 (레포별 언어 설정 적용)
→ 해당 레포의 초안 확인 및 수정
→ 포맷 선택 후 발행
→ SQLite에 레포별 기록 저장
```

---

## 모듈별 확정 사항

### 1. 레포지토리 관리

- 레포 **추가 / 수정 / 삭제** 가능
- 레포별 개별 설정
  - GitLab 또는 GitHub 선택
  - API 방식 또는 로컬 git 폴더 방식 선택
  - 보안 파일 제외 규칙 (레포별 완전 독립)
  - AI 요약 언어 (레포별 독립 설정)
- 레포 목록은 사이드바에서 빠르게 전환

### 2. 배포 감지

- **앱 실행 시** 등록된 모든 레포 자동 체크
- **수동 체크 버튼** (전체 / 특정 레포 선택)
- 새 커밋 감지 시 토스트 알림 (어느 레포인지 표시)
- 시작 프로그램 등록 옵션 (설정에서 켜기/끄기)

### 3. diff 소스 (레포별 설정)

| 방식 | 설명 |
|------|------|
| GitLab/GitHub API | 원격 저장소에서 직접 가져옴 |
| 로컬 git 폴더 | PC에 clone된 폴더 경로 지정 |

### 4. 보안 파일 제외 (레포별 완전 독립)

- 전역 규칙 없음
- 레포마다 완전히 독립된 규칙 설정
- 경로 패턴 예시: `**/.env`, `**/secrets/**`
- 확장자 패턴 예시: `.pem`, `.key`
- 레포 추가 및 레포 설정 화면에서 관리

### 5. AI 요약

- AI 제공자 및 API Key: 전역 설정 (Claude / GPT, 교체 가능)
- **요약 언어: 레포별 개별 설정**
  - 한국어만
  - 영어만
  - 한국어 + 영어 동시 (토큰 약 2배 소모 — 선택 시 경고 안내)
- 변경 유형 자동 분류: 버그수정 / 기능추가 / UI변경 / 성능개선

### 6. 로컬 저장 (SQLite)

| 저장 항목 | 범위 |
| ------ | ------ |
| 레포지토리 목록 및 설정 | 레포별 |
| 보안 파일 제외 규칙 | 레포별 |
| 초안 생성 기록 | 레포별 |
| AI 제공자 및 API Key | 전역 |
| Naver Works API Key | 전역 |
| 앱 설정 (언어, 테마, 시작 프로그램 등록) | 전역 |

### 7. 앱 UI

- 사이드바: 레포 목록 + 레포별 새 커밋 알림 뱃지
- 레포 선택 시 해당 레포의 초안 및 기록 표시
- 라이트 / 다크모드 전환
- 연록 ~ 초록 ~ 청록 사이의 컬러 (햇살이 들어오는 숲)
- 한 / 영 언어 전환
- 레포 추가 / 수정 / 설정 화면

### 8. 발행 포맷

| 포맷 | 동작 |
|------|------|
| Naver Works | HTML 클립보드 복사 (에디터 붙여넣기용) |
| 마크다운 | `.md` 파일 다운로드 |
| 일반 텍스트 | 클립보드 복사 |
